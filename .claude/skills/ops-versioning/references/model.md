# How versions work (plain English)

The file you are looking at is the live one. It is the current copy, the one that counts.
Everything below is just safety nets behind it.

When you save a version, the skill makes a frozen copy of the file as it is right now. You
never have to find that copy or know where it is kept. Each frozen copy is stamped with the
date and time and a short note, so you can tell them apart later.

Frozen copies are never changed and never deleted. The list only grows. So you can always
go back, and going back never throws anything away.

When you go back to an older copy, the skill saves the current file first, as a fresh frozen
copy, and only then puts the older one in its place. That means even an "undo" keeps a copy
of what you had, so you can change your mind again. Nothing is ever lost.

There is also a plain list of every saved copy, with its date, note, and size, oldest first.
You just ask to see your saved copies and you get the list. No tools, no special words.

One exception: websites and apps. A website or app is not a single file, it is a whole
project, and it goes live for real visitors. Those do not use frozen file copies. They use
the website tool that puts changes live, which keeps a private preview and a check before
anything goes live. If you ask to save or roll back a website, you get pointed to that tool,
because that is the safe way to roll a live site back.

One more thing so you are never caught out: these frozen copies live on this computer. They
are your fast, local history, and "go back" always works right here. The file you are working
on still gets saved and backed up the usual way, so it reaches your other machines; the
in-between frozen copies stay on this one. Nothing here ever deletes your work, it just keeps
the quick history local.
