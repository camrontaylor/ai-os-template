# Codex-Native Ad Creative Generation

This is the engine reference for `viz-ad-creative-codex`. It assumes the operator is running inside Codex and wants image generation from the subscription/native runtime, not from a model API key.

## Runtime Contract

- Do not ask for a generation API key in this variant. It runs from the native Codex image generation capability.
- Use the native Codex image generation capability through the available `imagegen` skill/tool.
- Save every prompt and every output to the campaign folder. Chat history is not the asset ledger.
- If native image generation is unavailable, stop before image generation and deliver the prompt pack plus a recommended manual/subscription path.

## Prompt Pack Shape

Save one prompt file per concept and ratio:

```text
projects/viz-ad-creative-codex/{client}/{date}_{campaign}/prompts/{concept}/{variant}_{ratio}.md
```

Each prompt file must include:

```markdown
# {Concept} - {Variant} - {Ratio}

## Goal
{platform, placement, awareness stage, metric}

## Exact Image Prompt
{single prompt to pass to image generation}

## Brand Lock
- Colors: {hex values}
- Typography feel: {font/style description}
- Logo rule: {include/exclude/placement}
- Product/reference rule: {what must stay visually consistent}
- Composition: {safe-zone and focal-point rules}

## In-Image Text
{exact text in quotes, or "None"}

## Negative Prompt
Avoid: generic stock-ad look, distorted hands, fake UI text, unreadable text, off-brand colors, extra logos, policy-risk claims, unrealistic before/after promises.

## QA Targets
- Ratio:
- Target pixels:
- Platform:
- Copy limit checked:
```

## Codex Image Generation Pattern

Use the native image generator once per approved creative/ratio. Pass:

1. The detailed prompt.
2. The requested ratio and nearest target dimensions from `references/ad-sizes.md`.
3. Any uploaded reference image, product image, or logo already available in the current Codex context.
4. A negative prompt, even when the tool does not expose a dedicated field; append it as "avoid..." inside the prompt.

Example prompt:

```text
Create a 4:5 paid social ad image for Meta feed, 1080x1350 target.
Brand: Acme Analytics. Use deep navy #101828, signal green #12B76A, white, clean B2B SaaS visual style, crisp editorial lighting, modern product screenshot feel.
Audience: operations leaders who are tired of manual weekly reporting.
Visual: laptop on a tidy desk showing an analytics dashboard, one highlighted KPI card, subtle human hand on trackpad, premium but not luxury.
Exact in-image text: "Reports in 5 minutes"
CTA button text: "Start free"
Keep all text inside the central 70% safe zone. Leave room for Meta UI overlays.
Avoid fake brand logos, unreadable UI labels, distorted hands, exaggerated income claims, and generic stock-photo smiles.
```

## Consistency Rules

- Reuse the same brand-lock block on every prompt in a batch.
- Change one variable at a time: hook, offer, visual metaphor, proof point, or CTA.
- Use at least two executions for each hook structure.
- Keep in-image text short. If exact typography matters, generate the image without baked-in headline text and finish it through `viz-ad-creative-figma`.
- Use the image generator for still creatives only. For motion or video, route to `viz-ad-creative-fal` or a subscription tool like Runway.

## Manual Subscription Fallbacks

When native generation is unavailable, the same prompt pack can be run manually in:

| Tool | Best use |
|---|---|
| ChatGPT Images | Low-friction subscription generation and image editing |
| Adobe Firefly | Commercial-safe edits, expand/remove/background workflows |
| Runway | Short video or motion concepts |
| Midjourney | High-aesthetic concept exploration, less deterministic brand control |
| Canva | Template assembly and simple social ad production |

Record the actual tool used in `slate.csv` even if the prompt was generated by this skill.
