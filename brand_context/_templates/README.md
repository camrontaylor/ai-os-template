# Brand Context Templates

These are blank structural templates. Each one shows the section headers a real brand file contains, so you can see what goes where. They are not the live files. Onboarding fills the real brand files into the parent `brand_context/` folder, not in here. Keeping these in this `_templates/` subfolder means the parent `brand_context/` stays empty, so the first-run onboarding check still fires for new users. You can safely ignore or delete this folder.
